## 项目经历

### 扫地机器人智能客服系统
**项目时间**：2023.10 - 2024.04 | **项目角色**：独立开发者 | **项目来源**：个人项目

**项目简介**：
为扫地机器人产品开发智能客服系统，用户可通过自然语言咨询产品使用、故障排查、获取个性化使用报告。系统基于ReAct模式，支持多轮对话和工具调用。

**技术架构**：
Python + LangChain + ChromaDB + Ollama + FastAPI

**主要工作**：

1. **ReAct Agent架构实现**
   - 采用ReAct（Reasoning + Acting）模式，实现「思考→行动→观察→再思考」的循环决策流程
   - 设计7个工具函数：RAG检索、天气查询、用户定位、用户ID获取、月份获取、外部数据获取、报告上下文注入
   - 实现工具描述和参数定义，确保模型正确理解工具用途和入参格式

2. **RAG知识检索模块**
   - 基于ChromaDB构建向量知识库，存储产品手册、故障案例、选购指南等文档
   - 使用RecursiveCharacterTextSplitter进行文本分片，配置chunk_size和chunk_overlap
   - 实现MD5去重机制，避免重复加载相同文档
   - 封装rag_summarize工具，支持根据用户问题检索相关资料并生成回答

3. **中间件与流程控制**
   - 实现monitor_tool监控工具调用、log_before_model记录模型输入、report_prompt_switch动态切换提示词
   - 设计报告生成流程：获取用户ID→获取月份→注入上下文→获取外部数据→生成报告
   - 支持流式输出，提升用户交互体验

4. **外部数据集成**
   - 解析CSV格式的用户使用记录，构建用户画像数据
   - 实现fetch_external_data工具，支持按用户ID和月份检索使用数据
   - 为个性化报告生成提供数据支撑

**项目成果**：
- 支持产品咨询、故障排查、个性化报告生成等多种场景
- 工具调用准确率稳定，响应时间<3秒
- 项目代码已托管GitHub，文档完整，可独立部署运行

**项目简介**：
基于LangGraph实现的多智能体协作系统，用于简历解析、评分、真实性验证和面试题生成。系统采用状态机编排工作流，支持条件分支和循环重试。

**技术架构**：
Python + LangGraph + ChromaDB + FastAPI + Ollama

**主要工作**：
