## 审核标准

请按以下标准逐一检查：

### 1. 技能真实性
- 声称"精通"某技能，必须有3年以上深度使用经验或高难度项目佐证
- 技能数量超过8项且缺乏深度项目支撑，标记为广度问题
- 技能描述与项目经验不匹配，标记为不实

### 2. 时间线一致性
- 工作时间重叠超过1个月，标记为时间线问题
- 工作间隔超过6个月无说明，标记为空窗期
- 项目时间超出工作时间范围，标记为不一致
- 2年内晋升超过2级，标记为可疑

### 3. 描述合理性
- 业绩提升数据超过200%且无具体方法说明，标记为夸大
- 职责描述少于10字或使用模板化语言（如"负责开发工作"），标记为模糊
- 项目复杂度与职级不匹配（如初级岗位主导大型架构），标记为可疑

### 4. 经验匹配度
- 工作年限与技能深度明显不符（如1年经验声称精通多项技术），标记为经验问题
- 个人贡献占比超过80%的大型项目，标记为夸大嫌疑

## 输出格式

```json
{{
  "is_valid": true,
  "risk_level": "低",
  "issues": [],
  "suggestions": ""
}}
```

### 字段说明：
- **is_valid**: 
  - `true`: 问题数 ≤ 2 且无高风险问题
  - `false`: 问题数 ≥ 3 或存在高风险问题（如技能造假、时间线造假）
- **risk_level**: 
  - "低": 0-1个问题，且为描述类问题
  - "中": 2个问题，或涉及技能夸大
  - "高": 3个及以上问题，或涉及造假嫌疑
- **issues**: 问题列表，无问题时为空数组
- **suggestions**: 审核建议，无问题时输出"简历信息完整，无明显问题"

### 输出示例：

**有问题的输出：**
```json
{{
  "is_valid": false,
  "risk_level": "高",
  "issues": [
    {{
      "type": "技能夸大",
      "description": "声称精通Python，但项目中未体现高难度Python开发",
      "evidence": "项目描述仅涉及简单CRUD操作，无复杂逻辑或性能优化"
    }},
    {{
      "type": "时间线问题",
      "description": "两段工作时间存在重叠",
      "evidence": "A公司2022.01-2022.06，B公司2022.03-2022.08，重叠3个月"
    }}
  ],
  "suggestions": "建议降低技能匹配度分数，重点考察Python深度能力，并核实工作时间真实性"
}}
```

**无问题的输出：**
```json
{{
  "is_valid": true,
  "risk_level": "低",
  "issues": [],
  "suggestions": "简历信息完整，无明显问题"
}}
```
